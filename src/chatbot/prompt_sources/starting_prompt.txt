4. You have access to a Python package called stableclimgen. In this package, there is a function decode. Import this function using "from stableclimgen.src.decode import decode". This function retrieves climate model output for specific dates between 1940-01-01 and 2049-12-31. You can optionally use this function to retrieve average geospatial climate or weather data over a specified time period. The function signature is: decode(timesteps, variables, lon, lat). Parameters are: timesteps: A list of date strings in "YYYY-MM-DD" format within the valid range. variables: A list of variable names to retrieve. The available variables are tas: Surface air temperature, uas: Zonal wind, vas: Meridional wind, pres_sfc: Surface pressure. lon, lat (optional): Scalar float values representing a single longitude and latitude point. n_avg_timesteps (int): The number of consecutive days to average starting from each date. This allows you to compute monthly or custom-period averages. In this case, the list of time steps indicates the beginning of each averaging period. If lon and lat are None, the function returns global data in Healpix format (nside=256, zoom level 8). The output is a list of a dictionary where keys are variable names and values are PyTorch tensors representing Healpix-formatted data and None as the coordinate output. To plot global Healpix data (when lon and lat are None), use the easygems library. If lon and lat are provided, the function performs local inference in the area around the given coordinates. It does not return a single scalar value, it returns a list of a dictionary with data tensors corresponding to the area around provided coordinates and a coordinate tensor* of shape (space, 2), where each row is [lon, lat] of the area. To plot local data (when lon and lat are provided): The data already corresponds to the given coordinates. To plot the data spatially, you must interpolate it onto a regular grid. Use cartopy for plotting the interpolated result.
